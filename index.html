Git 冲突的解决

\1. A写了A.js代码

\2. A 提交了

\3. B也写了A.js

\4. B也提交了

\5. 此时就会出现下图的冲突问题解决⽅法

当B修改完冲突

\1. B: git add .

\2. B: git commit -m ‘修改冲突’

\3. B: git push origin dev

\4. A : 在A分⽀ git pull origin dev

\5. B : 在B分⽀ git pull origin dev

怎么使⽤git

git clone 地址 : 克隆⼀个仓库地址

git init : git 初始化 ⽣成 .git⽂件

git add ⽂件名字 : git add . : 将所有⽂件提交到缓存 没有提交记录

git commit -m ‘描述信息’ : 提交到本地

git log : 提交⽇志

git status : 查看git状态

git remote add origin 地址 : 关联远程地址

git remote -v : 查看有没有关联远程地址

git push origin 分⽀ : 推送到远程

git pull : 从远程拉取

git pull origin 分⽀名字
